HTTPS流程

个人理解（未学过计算机网络 need todo）

设置SSL证书

上篇

  1. 进入阿里云控制台，点击SSL证书

  2. 进入后，点击左侧的SSL证书选项

  3. 再点击免费证书

  4. 点击立即购买

  5. 免费的证书每人每年只能拥有20张。所以选项默认不做变动。立即购买

  6. 点击创建证书

  7. 点击左侧栏的信息管理，新建联系人。按要求填写信息。

  8. 点击左侧栏SSL证书，点击免费证书，点击证书申请，填写域名，选择自动DNS验证，系统自动生成，下一步。

  9. 记录下这里的记录类型，主机记录，记录值

  10. 新网页标签打开控制台里的域名。

  11. 点击左侧栏【域名列表】，找到一个域名，点击右侧的【解析】。

  12. 点击添加记录。将9中的《记录类型》，《主机记录》，《记录值》选择或者填。《解析路线》和《TTL》默认。点击【确认】。

  13. 回到8中的页面点击【验证】，成功即可。点击【下载】将证书下载保存（选择Nginx版本）。

下篇

  1. 使用xhsell连接阿里云服务器

  2. 使用docker安装Nginx

    docker pull nginx

  3. 运行镜像

    docker run -p 80:80 -p 443:443 --name nginx -v /mnt/nginx/html:/usr/share/nginx/html -v /mnt/nginx/logs:/var/log/nginx -v /mnt/nginx/conf/nginx.conf:/etc/nginx/nginx.conf -v /mnt/nginx/cert/:/etc/nginx/cert/ --restart=always --privileged=true -d nginx

    前缀 /mnt/nginx 可以自定义。

  4. 运行完成后 docker ps查看下是否运行成功（是否存在，STATUS是不是up）

    然后进入cert文件夹

    cd /mnt/nginx/cert/

    将上篇第13步下载保存的压缩包拖动移动放入xshell窗口里（就是传输到服务当前文件夹，如果使用xftp更加方便）。

  5. 传输完成后。输入一下命令解压，可以得到后缀为pem和pub的文件。

    tar -zxvf 压缩包全名

  6. 使用xftp进入/mnt/nginx/conf/文件夹，右键nginx.conf文件，没有的话，自己创建一个。如下代码全部替换。然后保存。

    # ctrl + F 查找 填写 来寻找需要更改的地方。
worker_processes  1;

events {
    worker_connections  1024;
}

http {
    include       /etc/nginx/mime.types;      #不需要更改
    default_type  application/octet-stream;		#不需要更改

    log_format  main  '$remote_addr - $remote_user [$time_local] "$request" '
                      '$status $body_bytes_sent "$http_referer" '
                      '"$http_user_agent" "$http_x_forwarded_for"';		#不需要更改

    #access_log  logs/access.log  main;

    sendfile        on;		#不需要更改
    #tcp_nopush     on;

    #keepalive_timeout  0;
    keepalive_timeout  65;		#不需要更改

    #gzip  on;
	server {
    		listen 443 ssl; # 这里是新版Nginx的写法
    		server_name blogimg.ytte.top;    # 填写你的域名
    		# ssl on;        # 这种已经不使用了，这么写会报警告，可以直接去掉采用第一行的写法
    		root /home/git/hugo;   # 前台文件存放文件夹，一般使用 Nginx 初始化的文件夹，当然也可以自己修改 
    		index index.html;            # 上面配置的文件夹里面的index.html
   		ssl_certificate  cert/7448435_blogimg.ytte.top.pem;    #填写改成你的证书的名字
    		ssl_certificate_key cert/7448435_blogimg.ytte.top.key;#填写你的证书的名字
    		ssl_session_timeout 5m;
    		ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE:ECDH:AES:HIGH:!NULL:!aNULL:!MD5:!ADH:!RC4;
    		ssl_protocols TLSv1 TLSv1.1 TLSv1.2;
    		ssl_prefer_server_ciphers on;
    		location / {
     			index index.html index.htm;
   		}
	}
	server {
    		listen 80;
    		server_name blogimg.ytte.top;               #填写你的域名
    		rewrite ^(.*)$ https://$host:443$1 permanent;         # 把http的域名请求转成https且转发到443端口
	}
}

  7. 回到Xshell使用如下命令查看nginx的容器id，重启nginx是的配置文件生效。

    docker  ps 

docker restart nginx容器的id  

docker  ps  #查看是否重启成功（STATUS为up即成功，如果显示restarting，等一会再docker ps ，如果还显示restarting，可能是6中 nginx.conf 配置错误，仔细检查一下；如果还是不行，进入/mnt/nginx/logs下查看下error错误日志，在寻找解决办法）

  8. 至此https配置完成，我们只需要访问6中配置的域名（端口为80 默认不写），nginx就能帮我们把访问这个域名80端口自动转到这个域名443端口，实现https访问。

基于上述扩展

因为本例是基于上述的配置七牛云文件访问的二级域名。七牛云中使用的还是http方式，所以我们可以将其更改为https方式。

只需要重复二级域名操作 ——>2.配置七牛云中的步骤，在第7步与第8步之间，执行一下内容。

注意：加速域名要填写你申请ssl证书填写的域名

  • 加速域名填写不同于之前的二级玉米；通信协议选择【https】，点击【SSL 证书服务】

  • 点击【上传自有证书】。

  • 将本文上篇的第13步骤保存的压缩包解压，并把两个文件的内容复制这里（证书内容=>pem；证书私钥=>key）点击【确认上传】

  • 等待上传成功即可，返回添加域名界面，填写你申请ssl证书填写的域名，通信协议再次点击选择【https】，就可以选择已经添加的证书，之后执行二级域名中步骤（即第8步骤开始）。
